Cert Studio

本地 Root CA 管理与内部 HTTPS 证书签发工具，面向开发、内网服务和企业测试环境。

下载 Release 查看源码
当前交付能力

Root CA 创建/导入、系统信任导入、PFX 导出、Root CA 备份迁移、Release 自动打包和 PR 必过检查。

Root CA 管理

创建、导入、备份和迁移本地 Root CA，私钥优先写入系统密钥环，必要时回退到本地加密存储。

证书签发

支持 DNS SAN、IP SAN、PFX/PKCS#12、Nginx 配置和 Electron 接入说明导出。

工程化流程

通过 GitHub Actions、Dependabot、分支保护、模板和 Project 管理持续交付节奏。

快速开始

  1. 从 GitHub Release 下载当前平台安装包。
  2. 打开 Root CA 页面，创建或导入 Root CA。
  3. 按需将 Root CA 导入系统信任区。
  4. 在证书签发页面填写域名、IP、有效期和 PFX 密码。
  5. 导出证书束，并把 server.key 保存在受控位置。

安全边界

  • 不要提交或上传 Root CA 私钥、Root CA 备份包、PFX 密码或 server.key。
  • company-root-ca.crt 是公开证书，可以分发给需要信任该 Root CA 的客户端。
  • Cert Studio 适用于本地开发、内网和测试环境，不是公网 CA 服务。
